Project Overview

AI WiFi CAM is an open-source project that combines the ESP32-CAM with powerful AI processing capabilities. The system streams video from the ESP32-CAM over WebSockets/TCP to a PC, where real-time AI processing is performed using Python with OpenCV and various AI libraries.

The processed video, with AI detections overlaid, is then made available through a web interface that can be accessed from any device on the same network.

Key Features

  • Real-time video streaming from ESP32-CAM
  • Object detection using YOLOv4
  • Face detection and recognition
  • Pose estimation
  • Web interface for easy access
  • Adjustable AI model selection
  • Configurable detection confidence threshold
  • Snapshot capability
  • Performance statistics display

System Architecture

The AI WiFi CAM system consists of two main components:

  1. ESP32-CAM: Captures video and streams it over WebSockets/TCP to the PC. The ESP32-CAM is programmed with Arduino code that handles the camera configuration, WiFi connection, and video streaming.
  2. PC Software: Receives the video stream, performs AI processing, and serves the processed video through a web interface. The PC software is written in Python and uses libraries such as OpenCV, TensorFlow, and MediaPipe for AI processing.

The system is designed to be modular and extensible, allowing for easy addition of new AI models and features.

AI Models

The AI WiFi CAM system supports multiple AI models for different types of detection and recognition:

  • YOLOv4: A state-of-the-art object detection model that can detect 80 different object categories, including people, vehicles, animals, and everyday objects.
  • MediaPipe Face Detection: A lightweight face detection model that can detect faces in real-time with high accuracy.
  • MediaPipe Pose: A pose estimation model that can detect human body poses and track key body points.

For more information about the AI models, see the Q&A Document.
